# Physics Rosetta v1  
  
ID: PR1_01  
STATEMENT: It is forbidden for any physics interpretation to modify, weaken, override, or constrain any kernel rule or kernel admissibility condition.  
FORBIDS: kernel modification via physics overlay, physics-overlay-gated admissibility, physics-overlay-gated rejection, physics-defined kernel exceptions  
PERMITS: removable physics annotations, optional physics labels with no kernel force  
OPEN: minimal audit criteria for non-binding physics overlays  
  
ID: PR1_02  
STATEMENT: It is forbidden to treat any physics interpretation as true, correct, or preferred by default.  
FORBIDS: truth-by-physics-overlay, correctness-by-physics-overlay, preferred-interpretation-by-default, privileged physics lens  
PERMITS: multiple optional physics lenses with no default correctness status  
OPEN: admissible non-binding confidence annotations for interpretations (if any)  
  
ID: PR1_03  
STATEMENT: It is forbidden to treat physics terminology as granting semantic force to kernel tokens beyond what is explicitly declared within the physics overlay declarations.  
FORBIDS: implicit semantics from physics labels, meaning inflation by physics naming, semantic completion by physics overlay  
PERMITS: explicit physics-meaning declarations that remain non-binding to kernel artifacts  
OPEN: minimal schema for explicit non-binding interpretation declarations  
  
ID: PR2_01  
STATEMENT: It is forbidden to conflate physics-overlay namespaces with kernel namespaces; physics-overlay tokens must not be treated as kernel tokens.

PERMITS: explicit physics overlay namespaces, explicit mapping tables from kernel tokens to physics tokens  
OPEN: minimal namespace declaration schema  
  
ID: PR2_02

PERMITS: explicit mapping declarations with explicit scope  
OPEN: minimal mapping declaration schema

OPEN: admissible premises for partial inverses (if any)
