# A1_INTEGRATION_BATCH__ANCHOR_WITNESS_WORKFLOW__v1
Status: PROPOSED / NONCANONICAL / BOUNDED A1 INTEGRATION BATCH
Date: 2026-03-09
Role: Normalize anchor and regeneration-witness use for live A1 family emission

## 1) Purpose
This batch turns the current anchor / witness doctrine into a single A1-family workflow for the remaining live families.

Target outcome:
- family-level claims cite compact anchor surfaces instead of repeated raw-run paths
- emitted A1 family judgment uses regeneration witnesses when they exist
- rescue and continuation evidence is kept separate from head-promotion language

This batch does not:
- change runtime
- define a new storage doctrine
- treat anchor or witness surfaces as earned truth

## 2) Emission Order
For a live A1 family surface that needs to emit role placement or admissibility:

1. read the family-local campaign or control pack
2. read the current family anchor surface
3. read the regeneration-witness companion when present
4. emit one default admissibility block
5. emit one optional family-aware admissibility hint block
6. record any preserved tension where older family prose is more aggressive than the live anchor / witness read

Operational meaning:
- family pack = branch shape and local campaign logic
- anchor surface = current family-level scope, phase, and citation surface
- regeneration witness = retained triple or retained selector chain that keeps the live role judgment replayable enough for doctrine

## 3) Precedence Rule
When these surfaces disagree:

- do not smooth away the contradiction
- keep the contradiction explicit in prose
- emit the narrower live family read shared by the anchor and regeneration witness
- do not let older proposal-local optimism outrank the newer anchor / witness bottleneck read

Practical rule:
- anchor or witness proof of survival is not proof of promotion
- rescue existence is not rescue efficacy
- broad executable survival is not narrow landing

## 5) Family-Level Workflow Rules
### A) Default admissibility emission is now mandatory for active live families
If a family is live enough to guide selector behavior, it should emit:
- `executable_head`
- `active_companion_floor`
- `late_passengers`
- `witness_only_terms`
- `residue_terms`
- `landing_blockers`
- `witness_floor`
- `current_readiness_status`

### B) Family-aware hints are the compression surface, not the source of truth
Use hints to:
- keep selector output deterministic
- carry stronger family-local role placement
- forbid head promotion that the selector cannot infer safely

Do not use hints to:
- bypass anchor / witness evidence
- convert a witness into a head by local prose cleanup

### C) Anchor surfaces replace repeated raw-run citation at family level
Prefer family anchors whenever the point is:
- role placement
- family phase
- survival vs saturation
- passenger-only status
- rescue-phase existence

Only cite a raw run directly when:
- the exact report
- the exact cold-core file
- or the exact strategy artifact
is the point of the sentence

### D) Regeneration witnesses define the retained minimum proof
When a regeneration witness exists, emitted doctrine should preserve the witness chain it retains:
- memo witness
- cold-core or continuation witness
- strategy witness

Use that retained chain to keep the family judgment replayable enough for doctrine without restoring full transient memo trees.

## 6) Rescue And Merge-Fence Consistency
For the live entropy route:

- broad seed pressure stays broad
- later `path_build` narrowing is real
- rescue-phase execution is real
- rescue-phase efficacy remains the live bottleneck
- later-phase family fencing may constrain rescue
- seed-phase family fencing remains rejected

Operational consequence:
- do not retune seed-phase family fences just because rescue still stalls
- do not speak as if rescue presence solved frontier movement
- keep rescue continuation surfaces in bottleneck language, not closure language

## 7) Preserved Tensions
Keep these visible instead of smoothing them away:

- `pairwise_correlation_spread_functional` remains useful as a colder alias candidate, but still does not justify head promotion
- `entropy_production_rate` survives in broad executable mode, but still does not justify standalone head status
- rescue continuation proves rescue existence, but still preserves a bottleneck witness rather than a closure witness
- local family saturation on substrate enrichment is real, but it is not a reason to reopen broader enrichment inflation

## 8) Immediate A1 Use Rule
When a new live A1 family output is written for one of these families:

- cite the family anchor first
- add the regeneration witness companion if it exists
- emit the family default admissibility block
- emit the family-aware hint block
- end with one explicit "do not overclaim" sentence if the family is still passenger-side, rescue-side, or saturation-side
