## 5) Processing Mode
Run systematically, doc by doc, through the source roots.

Do not optimize for:
- “best docs first”
- one-pass final interpretation
- total completion in one sweep

Do optimize for:
- bounded batches
- compression
- contradiction preservation
- source linkage
- re-entry for later passes

## 6) Pass Model
This is a looping process, not a one-pass final extraction.

The same source docs may be revisited later for different extraction modes.

Examples of extraction modes:
- `SOURCE_MAP_PASS`
- `TERM_CONFLICT_PASS`
- `ENGINE_PATTERN_PASS`
- `MATH_CLASS_PASS`
- `QIT_BRIDGE_PASS`
- `PERSONALITY_ANALOGY_PASS`
- `CONTRADICTION_REPROCESS_PASS`

Rule:
- do not try to do every extraction type at once for every doc
- each bounded batch must declare its extraction mode

## 9) Big-Doc / Small-Doc Rule
For large dense source docs:
- allow multiple extracts by function:
  - structure extract
  - term/conflict extract
  - engine-pattern extract
  - downstream consequence extract

For many small related docs:
- bundle them into one bounded batch
- record full source membership in `MANIFEST.json`

## 10) Compression Rule
This is a compression surface.

Do not:
- copy whole source docs
- produce giant omnibus summaries
- dump full text for convenience

Do:
- compress
- cluster
- preserve contradictions
- keep outputs reusable for later passes

## 11) Mutation Rule
Do not:
- append-save into active A2 surfaces from this process by default
- rewrite system docs
- treat this surface as direct control memory

Only:
- produce bounded intake artifacts in this surface

Promotion from this surface into active A2 should happen later, selectively, under separate control.

Any thread following this process should report after each bounded step:
- current phase
- what was read

- whether to stay on the current model
- exactly how many more `go on` prompts to queue
- what the next `go on` will do

First bounded task for a fresh high-entropy intake thread:
- inventory the four source roots
- start from the first docs encountered in folder order
- process only the first bounded batch
- if `sims/` appears, split it into its own batch class
- produce the first batch artifacts in this surface

  - engine patterns
  - term/conflict extraction
  - math-class extraction
