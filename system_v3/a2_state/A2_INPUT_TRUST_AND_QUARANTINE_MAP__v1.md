## 2) Medium-Trust Inputs
Use for conflict mapping, historical clarification, and preserved tensions.

- archived A2 state docs
- upgrade extracts
- old boots and save kits
- old extraction audits
- historical thread saves
- external aligned systems and research traditions, but only as pattern sources for control/validation/anti-drift extraction; never as ontology imports
- outside engineering repos, but only through a bounded intake filter:
  - inspect for the engineering problem solved
  - extract strongest reusable pattern
  - record hidden assumptions
  - classify `borrow`, `retool`, `reject`, or `defer`
  - keep initial results in outer A2 processing layers, not kernel memory

## 3) Quarantined / High-Noise Inputs
Preserve as hypothesis material only. Do not promote directly into structural A2 understanding.

- fused TOE cosmology material
- heavily blended Grok synthesis
- broad social / Leviathan framework docs
- strongly teleological material presented as if already solved

## 4) Promotion Rule
Input strength does not equal earned truth.

Promotion order:
1. preserve
2. classify
3. source-bind
4. cross-check
5. hand to A1 as proposal fuel if still useful

Additional control-plane rule:
- doctrine-only policy files may shape A2 operating discipline
- they do not by themselves create new transport law or lower-loop semantics

Additional outside-repo rule:
- disposable worker systems, browser bridges, simulation frameworks, clustering tools, and validation stacks may be adopted for engineering help
- but they must arrive as bounded engineering-pattern extracts, not as imported worldview or hidden control logic

## 5) Rejection / Quarantine Reasons
Typical reasons to quarantine:
- fused multiple layers together
- overclaims authority
- collapses QIT-first entropy into thermodynamic literalism
- teleology presented as settled
- too noisy to support clear distillation
