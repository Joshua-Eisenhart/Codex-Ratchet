# DOC_INDEX_STATUS_CAUTION__v1
Status: PROPOSED / NONCANONICAL / OPERATIONAL CAUTION
Date: 2026-03-06
Role: Caution note for `doc_index.json` and related status fields

## Purpose
`doc_index.json` and similar A2 indexing surfaces use status fields for operational sorting and source tracking.

These status fields are:
- source-local
- operational
- indexing metadata

They are not by themselves earned ratchet truth.

## Key Rule
Do not let document index status labels outrank:
- lower-loop process
- SIM evidence
- graveyard / negative pressure
- current runtime reality

## Examples
Fields like:
- `source_local_status`
- `canon_status`
- `READ_ONLY_SOURCE`
- local `CANON` wording

may be useful for organizing inputs, but they do not settle truth status for the system.
