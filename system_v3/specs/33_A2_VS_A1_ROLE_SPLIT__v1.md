# A2_VS_A1_ROLE_SPLIT__v1
Status: DRAFT / NONCANON / ACTIVE PROCEDURE SURFACE
Date: 2026-03-11
Owner: current `A2` controller, current `A1` proposal layer, and ZIP-bound worker lanes

## Purpose

This note makes the current role split explicit in one place.

Main rule:
- broad refinery belongs to `A2`
- proposal generation belongs to `A1`
- translation and packaging are narrower `A1` roles, not excuses to blur `A2` and `A1`

## `A2` role

`A2` owns:
- high-entropy reading
- source mining
- contradiction preservation
- unresolved tracking
- queue routing
- residue classification
- anticipatory batch shaping
- handoff preparation for `A1`

`A2` may:
- imagine downstream thread roles
- prebuild candidate batches
- mark `A1` queue readiness

`A2` may not:
- claim proposal work has already been performed
- free-run as `A1`
- use proposal-style outputs as if they were lower-loop truth

## `A1` role

`A1` owns only bounded proposal-side work from explicit fuel.

Current split:

### `A1_ROSETTA`
- selected-portion stripping/translation
- overlay-to-ratchet-safe target reshaping
- anti-smuggling translation work

### `A1_PROPOSAL`
- branch families
- positive candidates
- negative classes
- rescue transforms
- explicit admissibility / blocker outputs

### `A1_PACKAGING`
- structuring already-generated proposal outputs for downstream packet/schema/runtime use
- fail-closed packaging

`A1` may not:
- perform broad source refinery
- replace `A2`
- self-authorize canon

## Refinery rule

Full-stack refinery is `A2`.

Meaning:
- broad reading
- mass distillation
- source-to-residue reduction
- large family triage
- role anticipation
- batch shaping

are all `A2` work.

Do not call this `A1 refinery`.

If `A1` touches refinement at all, it is only:
- selected translation work
- candidate-family tightening
- packet shaping

from already prepared bounded fuel.

## Trigger rule

`A1` may run only when:
- a valid `A1_QUEUE_STATUS` says it is ready, and
- the matching `A2 -> A1` handoff exists, and
- the matching `A1` boot exists

Otherwise:
- `A1_QUEUE_STATUS: NO_WORK`
or
- blocked status

## Handoff rule

`A2` hands `A1` only:
- explicit update notes
- explicit impact notes
- explicit deltas
- contradiction maps
- residue clusters
- rosetta mappings
- queue/routing notes
- ZIP packets already bound to `A1`

Not:
- raw ambient source mass
- hidden controller memory
- uncategorized theory spill

## Lower-loop boundary

`A0 / B / SIM` are not chat-role layers in the same sense.

They remain:
- code/runtime/contract governed
- downstream of `A1`

So the role stack is:
- `A2` refines and dispatches
- `A1` proposes
- lower loop evaluates and constrains

## Immediate implication

Current system gap closed by this note:
- role blur between `A2 refinery` and `A1` subroles is now explicitly fenced in current system language

Still pending:
- optional later split if `A1_PACKAGING` grows enough to need a separate tighter boot
