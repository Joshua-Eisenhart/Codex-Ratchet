## 4. Strict Candidate Menu

### 4.1 Shell-label cq bridge

\[
\Xi_{\mathrm{shell}}^{\mathrm{cq}}:x\mapsto \sum_r p_r(x)\, |r\rangle\langle r|_S \otimes \rho_{LR}^{(r)}(x)
\]

Current read:

- this is the strict pointwise shell candidate already closest to executable form
- the strict bakeoff surface uses this style of shell-label register construction
- as currently instantiated in shell-strata form, it is killed as the final pointwise shell answer

Verdict:

- `KILL` as the finished strict shell answer
- `KEEP` only as a scaffold pattern showing the right kind of explicit label/state factorization

Why it survives as a scaffold:

- it keeps the shell register explicit
- it avoids the old superposition shortcut
- it shows what a legitimate shell-aware bridge family has to look like structurally

Why it dies as the answer:

- current shell-strata instantiation is geometry-blind / loop-blind on the strict bakeoff surface
- it does not yet buy enough beyond relabeled pointwise bookkeeping

### 4.2 Boundary/interior shell-cut state family

\[
\Xi_{\mathrm{shell}}^{\partial/\mathrm{int}}:x\mapsto \{(r,w_r,\rho_{A_rB_r}(x))\}_r
\]

with the intended shell cut

\[
A_r|B_r = I_r|B_r
\]

Current read:

- this is the strongest doctrine-compatible shell bridge family
- it matches the current shell-cut owner packet directly
- it is not one density matrix but a shell-indexed family of cut states

Strengths:

- lands naturally in the admitted shell doctrine
- preserves the distinction between bridge and cut ownership
- gives a clean home for
\[
\Phi_0(\rho)=\sum_r w_r I_c(A_r\rangle B_r)_\rho
\]

Open burden:

- exact shell algebra for \(A_r|B_r\) is still missing
- exact construction of \(\rho_{A_rB_r}(x)\) from live geometry is still missing
- the typed shell cut contract now owns the minimum \((r,w_r,A_r|B_r,\rho_r)\) target that a strict shell bridge must land in
- the shell-algebra packet now owns the ranked microscopic option space for what \(A_r|B_r\) could mean without overclaiming closure

Verdict:

- `KEEP` as the strongest strict shell replacement family
- still `OPEN`, not closed

### 4.3 Shell-history hybrid bridge

\[
\Xi_{\mathrm{shellhist}}:h\mapsto \{(r,w_r,\rho_{A_rB_r}^{\,h})\}_r
\]

Current read:

- combines the strongest live executable bridge idea (`Xi_hist`) with the strongest doctrine-facing shell cut family
- gives shell structure a trajectory-window realization instead of demanding a purely pointwise shell theorem first

Strengths:

- inherits support from the surviving history family
- avoids forcing shell closure through the killed pointwise shortcut
- may be the cleanest executable route into a shell-weighted coherent-information family

Risk:

- can drift into “history does all the work, shell is cosmetic” if shell dependence is not explicit

Verdict:

- `KEEP` as a serious hybrid route
- below pure boundary/interior shell family doctrinally
- above naive pointwise shell replacement executably
