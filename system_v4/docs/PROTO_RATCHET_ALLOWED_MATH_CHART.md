# Proto-Ratchet Allowed Math Chart

**Date:** 2026-03-29
**Purpose:** Pre-ratchet chart for building up from root constraints into the geometry layer and the base allowed mathematics without prematurely collapsing canon, working, and proposal surfaces together.
**Scope discipline:** This file is a proto-ratcheting chart, not a closure claim. It preserves the distinction between:

- **canon**: hard locks and thin admissibility surfaces
- **working**: live QIT/geometry realization packets that currently fit the canon
- **proposal**: candidate or later-admission surfaces that remain open

---

## How To Read This Chart

The ordering here is intentional:

1. root constraints
2. early-allowed math
3. late-derived math
4. banned-as-primitive math

This is a proto-ratchet buildup order, not a final ontology claim. The goal is to keep the attractor basin high-resolution enough that later ratcheting can still kill, split, or preserve distinctions correctly.

### Root Handoff Rule

The thin-canon pair does directly earn:

- finite admissible carriers
- finite probes and witness discipline
- order-sensitive composition
- noncommuting operator admission
- the later manifold statement `C -> M(C)`

The thin-canon pair does **not** directly earn:

- a unique `C^2` realization
- a unique density-matrix ontology
- a unique `S^3` / Hopf / Weyl geometry
- a finished entropy kernel
- a finished bridge `Xi`

### Late-Derived Math Notes

| Note | Why it matters |
|---|---|
| Late-derived does not mean weak or optional. | Some of the strongest canon claims, like `M(C)` and `A_i : M(C) -> V_i`, are late-derived relative to the root constraints. |
| `S^3` / Hopf / torus geometry should stay visibly "derived geometry". | This is one of the easiest places for LLM smoothing to accidentally smuggle geometry in as primitive. |
| `Ax0` belongs here, not in the geometry layer. | Even when the geometry is real and executable, `Ax0` still needs a bridge into a cut state. |

### Late-Derived Admission Rule

Use the following escalation discipline:

| Move | What must already be earned |
|---|---|
| admissible core -> favored live math basin | executable finite-QIT state/operator/probe packet |
| favored live math basin -> favored live geometry basin | source-backed Hopf/Weyl equations plus real geometry probes |
| favored live geometry basin -> `Ax0` kernel and bridge | explicit cut-state target and executable bridge witnesses |
