# Proto Ratchet Axis 0 Dynamic Shell Sim Program

## 1. Purpose

Turn the dynamic discrete shell proposal into bounded discriminating sim lanes without promoting it to canon.

This packet is proposal-only. It defines what to test, not what is already true.

---

## 2. Lane A

**Dynamic shell versus static shell**

Question:

- does a discrete pulsing shell model separate meaningful `Axis 0` behavior from a static shell-partition model?

Minimal target:

- build a finite shell ladder
- compare static shell observables against shell observables updated under discrete inward / outward tick rules
- check whether nontrivial signal appears only when shell motion is included

Keep signals:

- dynamic shell updates produce nontrivial changes in shell-cut observables
- static shell version stays flatter or less discriminative on the same carrier

Kill signals:

- dynamic shell adds no measurable separation over static shell bookkeeping

---

## 3. Lane B

**Discrete finite shell ticks versus continuum-style shell gradation**

Question:

- is a finite tick-based shell ladder sufficient, or does the proposal only look meaningful when smuggled into a fake continuum?

Minimal target:

- define a finite shell index set
- evaluate shell-step finite differences
- test whether the intended bridge observables remain stable under finite tick updates

Keep signals:

- finite shell differences already carry the relevant bridge information
- no continuum limit is needed to see the effect

Kill signals:

- the proposal only works when treated as an effectively continuous shell field

---

## 4. Lane C

**Tensor-network shell reading versus ordinary cut bookkeeping**

Question:

- does a tensor-network or entanglement-layer reading give a sharper shell-bridge observable than ordinary nested-cut bookkeeping?

Minimal target:

- define shell layers as discrete cut layers
- compare cut entropy, mutual information, coherent information, or path-entropy style observables across those layers
- test whether compression / expansion language earns extra discrimination rather than only extra metaphor

Keep signals:

- tensor-layer observables sharpen shell discrimination or shell-history discrimination
- compression / expansion language maps to concrete finite observables

Kill signals:

- tensor-network language adds no executable separation over the current cut-family bookkeeping

---

## 5. Keep / Kill Signals

Keep:

- shell motion matters
- discrete shell ticks are sufficient
- tensor-layer observables give cleaner discrimination than static shell language
- history-shaped and shell-shaped bridges can be coupled without collapsing into raw geometry

Kill:

- shell language stays purely metaphorical
- shell motion does not improve discrimination
- continuum assumptions are secretly required
- tensor-network language adds vocabulary but no executable gain
