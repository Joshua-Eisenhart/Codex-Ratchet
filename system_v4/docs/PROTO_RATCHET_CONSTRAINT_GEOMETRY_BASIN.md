## 1. Proto-Ratchet Discipline

This packet treats the following as different layers:

1. root constraints
2. allowed mathematics
3. geometry buildup
4. `Axis 0` as a later functional on a cut-state bridge
5. executable evidence
6. overlay language

The core anti-smoothing rule is:

- do not treat constraints as geometry
- do not treat geometry as `Axis 0`
- do not treat a working bridge as canon
- do not treat overlays as kernel semantics

## 3. Root Constraints

### 3.1 Thin canon root pair

| Constraint | Exact or near-exact statement | Immediate consequence |
|---|---|---|
| `F01_FINITUDE` | finite encodings, bounded distinguishability, no completed infinities, decidable admissibility | finite-dimensional carriers, finite operator bases, finite Kraus families, finite axis codomains |
| `N01_NONCOMMUTATION` | order-sensitive composition, no swap-by-default, sequence belongs to the object | `AB != BA` in general, `A rho != rho A`, precedence matters, loop holonomy is admissible |

Immediate local witness of `N01`:

\[
[\sigma_x,\sigma_y]=2i\sigma_z,\qquad
[\sigma_y,\sigma_z]=2i\sigma_x,\qquad
[\sigma_z,\sigma_x]=2i\sigma_y
\]

Thin-canon stop rule:

- thin canon currently stops at `F01_FINITUDE`, `N01_NONCOMMUTATION`, `C`, `M(C)`, induced geometry on `M(C)`, and `A_i : M(C) -> V_i`
- thin canon does not yet include `S^3`, Hopf, Weyl sheets, Clifford torus, or any final `Ax0` bridge
- those later surfaces may be favored and source-backed, but they are not the same layer as the root pair

### 3.2 Deeper proto-foundational constraints

| Proto foundation item | Why it matters |
|---|---|
| constraint on distinguishability as the deep substance claim | keeps information, geometry, dynamics, and bookkeeping downstream of constraint rather than primitive |
| no primitive identity | blocks free objecthood before admissible probes |
| no primitive equality | blocks free substitution and unearned sameness |
| finite witness discipline | forces claims to remain executable and contractable |
| axes are discovered, not primitive | keeps later axis work ratchetable and killable |

### 3.3 Constraint-to-geometry handoff

| Object | Pure math | Status |
|---|---|---|
| constraint set | \(C=\{F01,N01,\text{admissible probe rules},\text{admissible composition rules}\}\) | thin canon |
| constraint manifold | \(M(C)=\{x:x\text{ admissible under }C\}\) | thin canon |
| geometry | compatibility structure induced by \(C\) on \(M(C)\) | thin canon |
| axis slice | \(A_i:M(C)\to V_i\) | thin canon |

Build order:

\[
\text{constraints} \to M(C) \to \text{geometry on }M(C) \to \text{axis slices}
\]

Do not smooth this into "canon geometry is already `S^3`/Hopf/Weyl".

### 3.4 Root-constraint to allowed-math handoff

This is the first ratchet hinge.

What the root pair directly forces:

| Root item | Earliest earned admission |
|---|---|
| `F01_FINITUDE` | finite carriers, finite operator registries, finite probe families, finite path encodings |
| `N01_NONCOMMUTATION` | order-sensitive composition, noncommuting operator algebra, guarded precedence, path dependence |

What the root pair does **not** force by itself:

| Not forced yet | Why it stays later |
|---|---|
| a specific Hilbert realization such as `C^2` | finitude permits it, but does not uniquely demand it |
| a specific geometry such as `S^3` / Hopf | geometry is induced later on `M(C)` |
| a specific bridge `Xi` | bridges are later interface objects, not root admissibility |
| a specific entropy kernel | scalar functionals are late-admitted |

So the safe Round 2 handoff is:

\[
F01 + N01 \Rightarrow \text{finite admissible carriers and order-sensitive algebra}
\]

not yet:

\[
F01 + N01 \Rightarrow S^3/\text{Hopf/Weyl directly}
\]

### 4.1a Thin-canon admissible core vs favored realization

| Layer | Strongest current read | Status |

### 4.2 Late-derived math

## 5. Geometry Buildup

The current best combined ladder is:

\[
\text{constraints} \to M(C) \to \{H, D(\mathbb C^2), \text{probes}, \sigma\text{-basis}\} \to S^3 \to S^2 \text{ via Hopf} \to T_\eta \to \{\gamma_{\text{fiber}},\gamma_{\text{base}}\} \to (\psi_L,\psi_R,\rho_L,\rho_R)
\]

### 5.1 Build ladder

| Step | Object | Pure math | Status | Note |
|---|---|---|---|---|
| 0 | root constraints | `F01_FINITUDE`, `N01_NONCOMMUTATION` | thin canon | geometry comes after constraints |
| 1 | constraint set / manifold | \(C,\ M(C)\) | thin canon | proper pre-geometry layer |
| 2 | allowed QIT base | \(H=\mathbb C^2,\ D(\mathbb C^2),\ O=O^\dagger,\ p_O(\rho)=\mathrm{Tr}(O\rho)\) | source-backed working layer | admissible math, not geometry yet |
| 3 | Pauli basis | \(I,\sigma_x,\sigma_y,\sigma_z,\sigma_\pm\) | source-backed working layer | common algebra for state and generator language |
| 4 | density / Hamiltonian coordinates | \(\rho=\frac12(I+\vec r\cdot\vec\sigma),\ H_0=n_x\sigma_x+n_y\sigma_y+n_z\sigma_z\) | source-backed working layer | bridge to the concrete qubit realization |
| 5 | spinor carrier | \(S^3=\{\psi\in\mathbb C^2:\|\psi\|=1\}\) | direct geometry | first true carrier-geometry rung |
| 6 | sheeted carrier realization | \(S_s^3,\ s\in\{L,R\}\) | thin-canon realization | current concrete realization inside `M(C)` |
| 7 | Hopf / Bloch reduction | \(\pi(\psi)=\psi^\dagger\vec\sigma\psi\in S^2,\ \rho(\psi)=|\psi\rangle\langle\psi|\) | direct geometry | `S^3 -> S^2` plus density reduction |
| 8 | Hopf chart | \(\psi_s(\phi,\chi;\eta)\) | working geometry | local torus/loop chart |
| 9 | torus foliation | \(T_\eta\subset S^3\) | direct geometry | one nested Hopf torus per `eta` |
| 10 | Clifford torus | \(T_{\pi/4}\) | direct geometry | explicit special torus |
| 11 | nested Hopf-tori family | \(\{T_\eta\}_{\eta\in[0,\pi/2]}\) | direct geometry | torus family inside `S^3` |
| 12 | Hopf connection | \(\mathcal A=-i\psi^\dagger d\psi=d\phi+\cos(2\eta)d\chi\) | direct geometry | separates fiber from base |
| 13 | fiber loop | \(\gamma_{\mathrm{fiber}}^s(u)=\psi_s(\phi_0+u,\chi_0;\eta_0)\) | direct geometry | universal carrier language |
| 14 | base loop | \(\gamma_{\mathrm{base}}^s(u)=\psi_s(\phi_0-\cos(2\eta_0)u,\chi_0+u;\eta_0)\) | direct geometry | universal carrier language |
| 15 | density-path split | \(\rho_{\mathrm{fiber}}^s(u)=\rho_{\mathrm{fiber}}^s(0),\ \rho_{\mathrm{base}}^s(u)=|\gamma_{\mathrm{base}}^s(u)\rangle\langle\gamma_{\mathrm{base}}^s(u)|\) | direct geometry | fiber is density-stationary, base is density-traversing |
| 16 | Weyl working layer | \(\psi_L,\psi_R,\rho_L,\rho_R,\ H_L=+H_0,\ H_R=-H_0\) | compiled working layer | useful, but not the most minimal canon rung |

### 5.2 Important geometry notes

| Note | Why it matters |
|---|---|
| density matrices and probes belong to allowed math, not geometry proper | stops QIT state language from being mistaken for the geometry itself |
| `fiber/base` is universal geometry language | `inner/outer` is later engine-indexed overlay language |
| the `L/R` split exists in two strengths | thin-canon sheeted realization and compiled Weyl working layer should stay separate |
| `Axis 0` is not part of this ladder | it comes later as a functional after a bridge is chosen |
| `S^3` / Hopf / Weyl should be treated as favored realization, not root admissibility itself | preserves the ratchet order for later re-testing |

### 5.3 Current geometry-branch read

| Geometry branch read | Status | Why this is the current read |
|---|---|---|
| `S^3` / Hopf / `T_eta` / fiber-base / Weyl | strongest favored working realization | best overlap of source-backed equations, executable probes, and current operator/state packet |
| Hopf/Weyl as a stable attractor basin | strongest open meta-read | matches the executable basin without overclaiming uniqueness |
| Hopf/Weyl as unique canon geometry | not earned | root constraints and allowed-math handoff do not yet force this uniquely |
| non-Hopf geometry branches | open but weak | still admissible under branch-space discipline, but currently underdeveloped and probe-poor |

So the basin should currently be read as:

\[
\text{roots} \to \text{allowed math} \to \text{favored Hopf/Weyl realization}
\]

not yet:

\[
\text{roots} \Rightarrow \text{Hopf/Weyl uniquely}
