# A1 From A2 Distillation

Use this skill when the task is A1 proposal construction and refreshed A2 inputs are available or required.

## Core rules

- A1 is proposal-only.
- A1 must be rebuilt from refreshed A2 understanding, not from thread memory or ad hoc repo impressions.
- Do not jump directly from source corpus or high-entropy material to A1 when A2 handoff is missing.
- Keep positive, negative, alternative, and rescue lanes explicit.
- Naming must not imply earned truth.

## Workflow

1. Confirm that the task is actually A1 work:
   - branch family generation
   - executable distillation
   - cartridge shaping
   - SIM-linked proposal shaping
2. Require explicit A2 inputs.
3. If A2 is stale or missing, stop and route back to `a2-brain-refresh`.
4. Extract from A2:
   - target families
   - contradiction maps
   - residue clusters
   - invariant locks
   - SIM obligations
5. Emit proposal-only A1 outputs with explicit lane structure:
   - steelman
   - alternative formalism
   - adversarial negative
   - boundary repair
   - graveyard rescue
6. Keep outputs traceable to the A2 surfaces used.
7. If the requested task is actually A2 mining or lower-loop work, emit an off-process flag instead of forcing A1 output.

## Required output properties

- explicit target family or path campaign
- explicit proposal-only status
- explicit positives and negatives
- explicit rescue or alternate branch
- explicit unresolved assumptions
- explicit SIM hooks or evidence obligations when relevant
- explicit lineage back to A2 inputs

## Guardrails

- Do not fabricate missing A2 topic packets or residue clusters.
- Do not let rosetta cleaning masquerade as executable readiness.
- Do not flatten contradictions inherited from A2.
- Do not let candidate admissibility be mistaken for executable closure.
- Do not produce lower-loop claims, canon claims, or evidence claims from A1.
