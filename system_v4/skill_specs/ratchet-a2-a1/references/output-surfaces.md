# Output Surfaces

## A2 outputs

A2 should maintain:
- system map
- loop/protocol map
- term/conflict map
- input trust/quarantine map
- A2->A1 distillation inputs
- unresolved tensions

A2 is:
- noncanon
- append-only
- contradiction-preserving
- preparation-only

## A1 outputs

A1 should maintain:
- executable distillation update
- target-family / path-family model
- negative class registry
- rescue / graveyard operator map

A1 is:
- noncanon
- proposal-only
- branch-family based
- SIM-aware
- graveyard-aware

## Semantic hygiene

Always distinguish:
- earned ratchet state
- proposal
- overlay
- fuel
- archived state
- history/method
- working understanding

Do not let:
- `CANON`
- `CANONICAL`
- `SOLE_SOURCE_OF_TRUTH`
- `canon_status`
- `canonical_term`

outrank lower-loop process.
