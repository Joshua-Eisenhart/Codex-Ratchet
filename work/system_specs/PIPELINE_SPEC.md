# Pipeline Spec (full cycle)

Status: NONCANON | Updated: 2026-02-18
Implementation: `ratchet_core/runner.py` (--full-cycle mode)

## End-to-end flow

```
A2 fuel_queue.json
    ‚Üì fuel_to_strategy() or LLM strategy
A1_STRATEGY ZIP (on disk)
    ‚Üì expand_strategy() + compile_batch()
A1_BATCH ZIP (on disk)
    ‚Üì EXPORT_BLOCK written to outbox
B evaluate_export_block()
    ‚Üì accept/reject/park each spec
State saved (state.json)
    ‚Üì _run_pending_sims()
SIM evidence ingested
    ‚Üì run_feedback()
FEEDBACK ZIP (on disk)
    ‚Üì rosetta + constraint surface updated
```

## Runner modes

| Flag | What it does | Uses A1? | Uses ZIPs? | Uses feedback? |
|------|-------------|----------|------------|----------------|
| `--init` | Create fresh state with 2 axioms | No | No | No |
| `--run-once` | A0 hardcoded ladder ‚Üí B evaluate | No | No | No |
| `--cycle` | A0 ladder ‚Üí B ‚Üí SIM | No | No | No |
| `--full-cycle` | A1 strategy ‚Üí expand ‚Üí B ‚Üí SIM ‚Üí feedback | **Yes** | **Yes** | **Yes** |

## Full cycle steps (runner.py run_full_cycle)

1. **Load state** from run_dir/state.json (or fresh KernelState)
2. **Get strategy**: from `--strategy <file>` (LLM-produced) or `fuel_to_strategy()` (deterministic fallback)
3. **ZIP**: write A1_STRATEGY to run_dir/zips/
4. **Expand**: `expand_strategy(strategy, state)` ‚Üí list of (spec_id, B-grammar lines)
5. **Compile**: `compile_batch(items, state)` ‚Üí EXPORT_BLOCK string
6. **ZIP**: write A1_BATCH to run_dir/zips/
7. **Write outbox**: EXPORT_BLOCK text file in run_dir/outbox/
8. **B evaluates**: `kernel.evaluate_export_block(block, state)` ‚Äî accepts, rejects, parks
9. **Save state**
10. **Run sims**: `_run_pending_sims()` for specs needing evidence
11. **Save state** again
12. **Feedback**: `run_feedback(state)` ‚Äî updates rosetta.json + constraint_surface.json
13. **ZIP**: write FEEDBACK to run_dir/zips/
14. **Print summary** JSON to stdout

## CLI for the A1 workflow

```bash
# Generate briefing for the Cursor LLM
python3 a1_llm.py --briefing --run-dir runs/my_run

# LLM produces strategy_01.json (saved by the Cursor assistant)

# Validate strategy format
python3 a1_llm.py --validate runs/my_run/strategy_01.json

# Run the full cycle
python3 runner.py --full-cycle --strategy runs/my_run/strategy_01.json \
    --run-dir runs/my_run --sim-cap 5

# Check results ‚Äî re-run briefing
python3 a1_llm.py --briefing --run-dir runs/my_run
```

## Key parameters

| Parameter | Default | What it controls |
|-----------|---------|-----------------|
| `--sim-cap` | 3 | Max sims to run per cycle |
| `--max-entries` | 20 | Fuel entries to read (deterministic mode) |
| `--max-items` | 1000 | Max items from expansion |
| `--loops` | 1 | Number of cycles to run |
| `--strategy` | None | Path to LLM-produced strategy JSON |
| `--fuel` | None | Path to fuel_queue.json override |

## ZIP trail

Every cycle produces 3 ZIPs in run_dir/zips/:
- `A1_STRATEGY_<hash>.json` ‚Äî the strategy that was used
- `A1_BATCH_<hash>.json` ‚Äî metadata about the expanded batch
- `FEEDBACK_<hash>.json` ‚Äî what B accepted/rejected, rosetta changes

The ZIP trail IS the save system. No separate save operation needed.

## Graveyard-first architecture

Everything starts DEAD in the pool. `state.pool` is seeded from `fuel_queue.json` at init.
Each pool concept tracks: status (DEAD/ATTEMPTING/RESURRECTED), source doc, label, body,
chain_needs, and full attempt history with B rejection reasons.

A1's job: pick DEAD pool items to try resurrecting. Build admission chains bottom-up.
When B rejects an attempt, the reason is logged. A1 reformulates and tries again.
When something passes B + SIM, it moves to RESURRECTED.

The constraint ladder docs are FUEL for A1 ‚Äî high-quality ore (E0-E1 tier) but presumed
to have errors. A1 processes them: strips jargon, reformulates into B-grammar, lets B
find the smuggled assumptions.

## Invariants

- Graveyard ratio must stay >= 50% (from real alternatives, not garbage probes)
- Every TERM_DEF gets a paired positive SIM_SPEC + negative SIM_SPEC
- S_L0_MATH is auto-seeded if missing (foundation for all specs)
- 1 probe per 10 specs is auto-generated by compile_batch
- Each graveyard item must be structurally tied to a specific term
- Sims are tiered: small ‚Üí compound ‚Üí master sim (the endgame proof)
